# Trouble Log

## 1. 문제 요약

- 제목: AI 요약 기능 타임아웃 및 모델 미지원 404 에러
- 프로젝트: BidMatch
- 영역: 외부 연동 / AI 파트
- 심각도: High
- 상태: 해결됨
- 발견 시점: 2026-06-16
- 담당자: 김종록

## 2. 발생 상황

1. 공고 상세 페이지에서 'AI 핵심 3줄 요약 받기' 기능 호출 시, 일정 시간이 지나면 프론트엔드에서 연결을 끊어버리는 타임아웃 현상 발생.
2. 배포 시 `404 models/gemini-2.5-flash is not found` 와 같은 에러가 반환되며 요약 기능이 완전히 동작하지 않는 현상 발생.

## 3. 에러 메시지 또는 증상

- 화면: 'AI 요약 요청 중 오류가 발생했습니다.' 에러 팝업.
- 로그: `404 models/gemini-2.5-flash is not found`

## 4. 재현 방법

- 공고 상세 페이지 접속 후 '요약 받기' 버튼 클릭

## 5. 원인 분석

1. **타임아웃:** 생성형 AI 모델이 방대한 공고 원문을 분석하고 요약하는 데 통상 10~15초 이상이 소요되나, 프론트엔드 Axios 인스턴스의 글로벌 기본 타임아웃이 10초로 짧게 설정되어 있어 응답 전 연결 강제 종료됨.
2. **404 모델 에러:** 구글 제미나이(Gemini) API의 버전이나 API 키 권한 수준에 따라 특정 하드코딩된 모델명에 대한 접근이 거부됨.

| 가설 | 확인 방법 | 결과 |
| --- | --- | --- |
| Axios 타임아웃이 너무 짧다 | 프론트엔드 API 설정 확인 | 글로벌 timeout 10초로 확인됨 |
| 하드코딩된 모델명이 현재 키로 접근 불가하다 | API 공식 문서 및 키 권한 확인 | 지원 중단되거나 권한 없는 모델명 하드코딩 확인됨 |

## 6. 해결 방법

타임아웃 상향 및 동적 모델 탐색 로직 적용.

- 수정 파일: `NoticeDetail.jsx` (프론트엔드), `ai_service.py` (백엔드)
- 수정 내용: 
  1. (프론트엔드) AI 요약 API 호출부만 국소적으로 Axios `timeout` 옵션을 60000(60초)으로 상향.
  2. (백엔드) `genai.list_models()`를 통해 현재 API 키가 접근 허용된 모델 목록을 사전 조회하여 동적으로 모델(`flash` 계열 최우선)을 선택하도록 로직 개편.

## 7. 검증 결과

타임아웃 없이 정상적으로 응답을 대기하며, 동적 모델 선택으로 404 에러 없이 3줄 요약이 안정적으로 반환됨.
