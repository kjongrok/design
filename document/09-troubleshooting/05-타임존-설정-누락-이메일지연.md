# Trouble Log

## 1. 문제 요약

- 제목: 서버 환경 Timezone 설정 누락으로 인한 예약 작업(이메일 발송) 지연 버그
- 프로젝트: BidMatch
- 영역: 백엔드/스케줄러
- 심각도: High
- 상태: 해결됨
- 발견 시점: 2026-06-15
- 담당자: 김종록

## 2. 발생 상황

매일 오전 9시에 사용자 맞춤 공고 메일이 자동 발송되도록 APScheduler로 예약 작업(`hour=9`)을 설정해 두었으나, 클라우드타입(Cloudtype)에 배포한 이후 지정된 시간에 메일이 발송되지 않는 현상 발생.

## 3. 에러 메시지 또는 증상

에러는 없으나 지정 시간(KST 오전 9시)에 메일 발송 로직이 실행되지 않고 9시간 뒤에 발송됨.

## 4. 재현 방법

- 클라우드 배포 서버에서 APScheduler 구동 후 KST 오전 9시 발송 대기

## 5. 원인 분석

로컬 환경과 달리 클라우드 배포 서버(OS)의 기본 시간대(Timezone)가 한국 시간(KST)이 아닌 협정 세계시(UTC)로 설정되어 있었음. 스케줄러에 별도의 Timezone을 명시하지 않아, 서버가 `hour=9`를 UTC 기준 오전 9시(한국 시간 기준 오후 6시)로 인식하여 실제 메일 발송이 9시간 지연 대기되는 것이 근본 원인.

| 가설 | 확인 방법 | 결과 |
| --- | --- | --- |
| 서버의 OS 기본 타임존이 KST가 아닐 수 있다 | 클라우드 서버 시간 확인 | UTC로 설정됨을 확인 |

## 6. 해결 방법

APScheduler 초기화 시 명시적으로 타임존 지정.

- 수정 파일: `scheduler.py`
- 수정 내용: `BackgroundScheduler` 초기화 코드에 `timezone="Asia/Seoul"` 파라미터를 명시적으로 추가하여, 서버 운영체제의 기본 시간 설정에 의존하지 않고 무조건 한국 시간(KST)을 기준으로 실행되도록 수정.

클라우드 배포 서버에서도 정상적으로 KST 오전 9시에 이메일 발송 예약 작업이 실행됨.
