# Trouble Log

## 1. 문제 요약

- 제목: 공공데이터 API 응답 스펙 불일치 문제
- 프로젝트: BidMatch
- 영역: 데이터 수집 (백엔드)
- 심각도: High
- 상태: 해결됨
- 발견 시점: 2026-06-13
- 담당자: 변영진

## 2. 발생 상황

나라장터 OpenAPI를 연동하여 수집(Scraping)을 진행할 때, 공고의 '마감일' 데이터가 계속해서 `NULL`로 수집되거나 화면에 노출되지 않는 현상 발생.

## 3. 에러 메시지 또는 증상

마감일 데이터가 DB에 `NULL`로 적재되고 대시보드 화면에 노출되지 않음.

## 5. 원인 분석

조달청 공식 오픈 API 가이드 문서와 ERD 상으로는 마감일 키값이 `bidEndDt`일 것으로 예상했으나, **실제 API 응답의 JSON Payload를 분석해본 결과 `bidClseDt`라는 다른 키값으로 데이터가 넘어오고 있었음.**

| 가설 | 확인 방법 | 결과 |
| --- | --- | --- |
| 문서와 실제 응답 스펙이 다를 수 있다 | API 응답 JSON 직접 확인 및 디버깅 | `bidEndDt` 대신 `bidClseDt`로 반환됨 |

## 6. 해결 방법

백엔드 데이터 수집 스크래퍼의 데이터 매핑 로직을 즉각 수정.

- 수정 파일: `g2b_scraper_service.py`
- 수정 내용: 파싱 키값을 `bidEndDt`에서 `bidClseDt`로 변경

## 7. 검증 결과

모든 공고의 마감일 데이터가 정상적으로 DB에 적재되고 화면에 노출됨.
