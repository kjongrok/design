# Trouble Log

## 1. 문제 요약

- 제목: 프론트엔드 비동기(Promise) 에러 핸들링 누락
- 프로젝트: BidMatch
- 영역: 프론트엔드
- 심각도: Medium
- 상태: 해결됨
- 발견 시점: 2026-06-14
- 담당자: 김혜라

## 2. 발생 상황

관리자 대시보드에서 잘못된 조작(예: 자기 자신 삭제 시도)을 했을 때 백엔드에서는 정상적으로 400 Bad Request 에러와 친절한 안내 메시지를 내려주었지만, 화면에는 아무런 경고창이 뜨지 않고 무반응인 현상 발생.

## 3. 에러 메시지 또는 증상

브라우저 개발자 도구 콘솔에만 에러가 노출되고, 실제 사용자 UI/UX 측면에서는 동작 실패에 대한 피드백(Alert 등)이 전혀 노출되지 않음.

## 5. 원인 분석

Axios 통신 시 HTTP 상태 코드가 200번대가 아니면 `.catch()` 블록으로 빠지는데, 기존 코드에서는 `catch(err => console.error(err))`로 단순히 콘솔에만 에러를 찍고 화면으로 메시지를 띄워주는 로직이 누락되어 있었음.
