# BidMatch 요구사항 ID 최종본

## 1. 문서 목적

본 문서는 BidMatch의 요구사항 분석표와 화면 설계서를 연결하기 위한 최종 기능 ID 목록이다.

- 페이지 ID는 화면 단위로 관리한다.
- 요구기능 ID는 페이지 내부의 기능 단위로 관리한다.
- 화면의 `A1`, `D1`, `V1` 표시는 스토리보드 설명용으로 사용한다.
- AI가 적용되는 기능은 `AI`, `AI/RAG`, `AI/Rule`, `AI 개선 데이터`로 구분한다.

## 2. ID 작성 기준

| 구분 | 작성 형식 | 예시 |
|---|---|---|
| 페이지 ID | 기능약어-번호 | MAIN-001, LS-001, DT-001 |
| 요구기능 ID | 기능약어-F번호 | MAIN-F01, LS-F03, DT-F05 |
| 화면 라벨 | A/D/V + 번호 | A1, D1, V1 |

## 6. 제외 기능

다음 기능은 최종 요구사항에서 제외한다.

- 자격진단 검수 큐
- 관리자 자격진단 결과 수정
- 관리자 임명
- 회원 가입 유형 표시
- 서비스 연결 상태
- 기업 증빙 삭제

## 7. 작성 시 주의사항

- 요구사항 분석표의 ID에는 페이지 ID가 아니라 요구기능 ID를 적는다.
- 스토리보드 상단에는 페이지 ID를 적는다.
- 스토리보드 내부의 `A1`, `D1`, `V1` 표시는 기능 ID와 별도로 유지한다.
- 고객센터 화면 ID는 현재 작성된 `CS-001`, `CS-002`를 사용한다.
- 자격진단 검수 큐와 관련된 화면 및 요구사항은 추가하지 않는다.
