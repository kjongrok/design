# 📝 기획(PRD/ERD) 대비 실제 구현 변경점 분석 보고서

본 문서는 초기 해커톤 기획 단계에서 작성된 PRD(v1) 및 ERD와 **실제 구현된 최종 산출물 간의 주요 변경점과 확장된 기능들**을 분석한 자료입니다. 발표 자료(PT) 작성 시 핵심 차별화 포인트로 활용하시기 바랍니다.

---

## 1. 회원 인증 및 권한 관리 (Auth & Users)

> [!TIP]
> **핵심 변경점:** 단순 이메일 로그인을 넘어 소셜 로그인 도입 및 관리자(Admin) 권한 체계 신설

* **소셜 로그인(OAuth) 확장 도입**
  * **기획(PRD):** 이메일/비밀번호 기반의 자체 회원가입 및 JWT 인증만 설계 (FE-01, FE-02)
  * **실제 구현:** 사용자 편의성 극대화를 위해 **Google 및 Kakao 소셜 로그인**을 추가 구현.
  * **DB 변경점:** `users` 테이블에 소셜 연동을 위한 `auth_provider`(google, kakao 등) 및 `oauth_id` 컬럼 추가.
* **관리자(Admin) 권한 및 계정 상태 관리 추가**
  * **기획(PRD):** 일반 사용자(User) 중심의 서비스 로직만 존재.
  * **실제 구현:** 최고 관리자 개념 도입. 관리자가 악성 유저를 차단할 수 있는 기능 추가.
  * **DB 변경점:** `users` 테이블에 `role` (ADMIN, USER, COMPANY) 및 `status` (active, blocked) 컬럼 추가.

## 2. 기업 증빙 및 서류 관리 (Company Verification)

> [!IMPORTANT]
> **핵심 변경점:** 기업 신뢰도 확보를 위한 관리자 직접 승인 프로세스 추가

* **기업 인증 절차 고도화**
  * **기획(PRD):** `COMPANY_PROFILES`에 사업자등록번호 등 기본 정보만 저장.
  * **실제 구현:** 가입 시 증빙 서류를 제출하고, 관리자가 대시보드에서 이를 확인한 후 **승인(APPROVED) 또는 반려(REJECTED)** 처리하는 프로세스 구현.
  * **DB 변경점:** `company_profiles` 테이블에 `is_verified`(boolean) 및 `verification_status`(VARCHAR) 컬럼 신설.

## 3. 관리자 대시보드 (Admin Dashboard) 신설

> [!NOTE]
> **핵심 변경점:** 서비스 운영 및 모니터링을 위한 백오피스(Back-office) UI 전면 추가

* **기획(PRD):** 서비스 사용자를 위한 프론트엔드 뷰만 존재.
* **실제 구현:** 실시간 운영 현황을 파악할 수 있는 강력한 관리자 전용 React 대시보드 구축.
  1. **실시간 통계 모니터링:** 누적 가입자 수, 활성 조건 수, 수집된 공고 수, 메일 발송 현황 시각화.
  2. **수동 배치 실행:** 스케줄러를 기다리지 않고 관리자가 즉시 공고 수집 배치를 실행할 수 있는 기능.
  3. **통합 회원 관리:** 가입된 모든 회원의 상태(정지/활성) 변경 및 관리자 임명/해임, 계정 영구 삭제 기능.

## 4. 데이터베이스 및 스크래핑(배치) 로직 최적화

> [!WARNING]
> **핵심 변경점:** 실제 공공데이터 API 스펙의 차이를 반영한 안정화 작업

* **마감일(Deadline) 파싱 오류 수정**
  * **발생 원인:** 초기 기획 시 나라장터 OpenAPI 문서의 마감일 키값을 `bidEndDt`로 예상했으나, 실제 응답 데이터는 `bidClseDt`로 반환됨.
  * **실제 구현:** 배치 스크래퍼(`g2b_scraper_service.py`)가 실제 API 키값에 맞게 `bidClseDt`를 매핑하도록 긴급 수정하여 데이터 누락(NULL) 문제 해결.
* **관심 키워드 모델링 정교화**
  * **기획(ERD):** `USER_MATCH_RULES`에 포함/제외 키워드를 text로만 관리.
  * **실제 구현:** 단일 대표 키워드 저장을 위한 `keyword` (VARCHAR) 컬럼 추가로 매칭 검색 속도 및 쿼리 편의성 증대.

## 5. 운영 및 디버깅 단계 추가 고도화 (최신 변경점)

> [!NOTE]
> **핵심 변경점:** 실제 서비스 테스트 및 운영 중 발견된 문제 해결과 편의성 극대화를 위한 기능 추가

* **사용자 대시보드 통계 데이터 동적 연동**
  * **기획(PRD):** 단순 UI 형태로 목업(Mock-up) 데이터(하드코딩) 제공.
  * **실제 구현:** 사용자별 맞춤 공고 데이터와 미확인 알림 횟수 API를 연동하여, "오늘 마감 공고, 전체 매칭 공고, 마감 임박, 미확인 알림" 4개의 핵심 통계를 실시간으로 계산해 보여주도록 고도화.
* **비밀번호 재설정 기능 완전 자동화 신설**
  * **기획(PRD):** 초기 로그인 페이지에 비밀번호 찾기/재설정 기능 누락.
  * **실제 구현:** 가입된 이메일을 통해 8자리 무작위 임시 비밀번호를 발급하고, 이를 암호화하여 DB에 갱신한 뒤 즉시 이메일(SMTP)로 발송하는 전체 사이클(API 및 UI) 신설.
* **AI 요약 모델 동적 탐색 및 타임아웃 안정화**
  * **발생 원인:** 고정된 AI 모델명(`gemini-2.5-flash` 등) 사용 시 구글 API 권한 및 버전에 따라 404 모델 미발견 오류 발생, 또한 긴 요약 응답 시간으로 인한 프론트엔드 통신 끊김(Timeout).
  * **실제 구현:** `genai.list_models()`를 통해 서버 환경에 부여된 API 키가 허용하는 최적의 모델(우선순위: flash)을 알아서 동적 탐색하여 꽂아 넣도록 자동화하고, 해당 요청에 한해 타임아웃을 60초로 넉넉하게 연장하여 오류 원천 차단.
* **UI/UX 직관성(가시성) 개선 및 캘린더 고도화**
  * **실제 구현:** 경계선이 모호했던 기존 컴포넌트들을 뚜렷한 박스형 패널(`.panel`) 구조와 그림자(Shadow), 어두운 배경 대비로 변경하여 가시성을 크게 높였고, 캘린더(달력)의 그리드 구분선을 명확히 개선.
* **이메일 스케줄러 시간대(Timezone) 고정**
  * **발생 원인:** 서버가 배포되는 클라우드 환경(UTC)에 따라 오전 9시 정각 발송 스케줄이 어긋나는 문제.
  * **실제 구현:** `APScheduler`의 시간대를 명시적으로 한국 시간(`Asia/Seoul`)으로 고정하여 정확성 보장.
